涉及语言介绍

由于实际工作中总会有大量重复的数据处理工作, 因此会一门脚本语言, 批量完成数据处理任务是必不可少的. 在生物信息工作中有用到的编程/脚本语言非常多: linux发行版默认搭载的Bash, 过去用的比较多的Perl, 越来越火的Python, 绘图和单细胞领域用的非常多的R, 难学但是效率高的C++和java, 甚至是新兴的Go和Julia.

我曾经在研究生期间进行自学的时候就层纠结过...到底应该选那个入门比较好. 现在工作了两年之后回看起来...这些语言其实本质上都是工具而已. 既然是工具, 那么用什么不是最重要的, 只要能完成任务, 并且满足效率要求的, 都是可考虑的选择. 因此在入门的阶段, 随便选择一项, 了解基本的编程概念并完成一些简单任务即可.

而如果涉及到生产工作, 考虑的东西就比较实际了. 一是尽量使用工作团队中大多数人用的语言. 因为虽然编程的理念都差不多, 但是语言和语言间的特性还是差别明显的, 比如在Python中很简单就能完成的东西在Perl中不一定有对等物, 在R中使用多进程比较简单但是Python中就需要花点时间了解其多进程使用方式等等...这些或大或小的差异会使使用不同语言的人比较难以互相协助. 这既不利于个人学习, 也不利于合作交流. 另外一点则是你主攻项目所使用的语言了. 比如我曾经需要复刻一个序列处理程序, 但是由于其源代码使用Scala撰写, 我完全看不懂而无法进行. 从这个角度讲, 对之前提到的所有语言都有基本的语法了解是非常必要的...

那么总的来说, 入门就选简单易学且大家用的多的, 而开始实际工作的话, 主力语言选择团队主流语言, 然后在闲暇之余尽量多掌握一些其他常用语言的基础知识比较好.

那么需要了解的语言可能有哪些呢?

Bash

本质是大多数linux发行版(MacOS也是)默认搭载的shell, 其优势也在默认搭载上, 登入系统即用. 其应用一般是处理即写即抛的简单任务. 另外在构建流程中, 利用标准输入和标准输出将不同的程序串起来, 减少对磁盘的读写以提高运行效率.

Perl

听前辈说过去是生物信息领域最常用的语言, 我在硕士期间有自学过. 其具有很强的字符串处理能力, 同时有完备的生物信息模块支持. 但是在用惯了Python后, 缺失觉得它学起来比较难, 同时语言的中文资料有点缺乏(生物信息领域).

Python

简单易学的脚本语言, 不止是在生物信息领域, 在各个领域都有广泛的应用. 因为其应用广泛, 好用的第三方模块也相当多, 用来快速开发效率要求不高的程序非常合适. 与Perl相比中文材料非常丰富. 当然...其缺点也非常明显, 就是语言效率相当低下. 虽然有Cython这种解决方案, 但是这要求使用者会C++, 无疑提高了学习难度.

R

原本是在数理统计领域应用很多的语言(开发初衷也是服务于统计), 后来慢慢的就被扩展到了更多的领域. 在生物信息领域也有人专门建立的网站收集相关的分析模块. 目前在单细胞领域, 绝大部分主流的分析工具都是R包.

Go

Go语言来自Google, 是一门相对易学效率也很出色的语言. 目前其虽然有一些生物信息的模块, 但是目前来看应用还较少. 对于效率不敏感的项目使用Python/Perl, 对于效率敏感的项目或者部分上C++/Java依然是最常见的做法.

Julia

比Go还要年轻的语言, 号称要汇集C/Python/R的优点于一身. 目前我对Julia的了解不多, 其在生物信息领域的应用也还比较少(Biojulia还不完善, Opengene因为语言版本更迭无法使用), 可以了解并观望.

C++/Java

很多大型项目或者效率要求高项目的首选, 但是上手难度相对高, 开发速度也相对慢.
